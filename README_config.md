# Configuration Documentation

This document provides an overview of the configuration settings available in the `config.json` file. Each section is explained in detail below.

---

## General

The `general` section contains general configuration settings.

- **log_level**: Defines the logging level (e.g., `DEBUG`, `INFO`, etc.).
- **is_production**: Boolean flag to indicate if the application is running in production mode.

---

## Flight Controllers

The `drone` section defines settings related to flight controllers.

### Properties

- **properties**: A list of settings for each flight controller.

#### Example: `properties[0]`

- **URI**: Connection string address of the flight controller.
  - **tcp**: TCP connection string (e.g., `tcp:127.0.0.1:14550`).
  - **udp**: UDP connection string (e.g., `udp:127.0.0.1:5760`).
- **FLIGHT_MODES**: A mapping of flight mode codes to their respective names.

---

## Requests

The `requests` section contains settings related to making requests.

- **ping_check_by_host**: The host used to check if the network is connected in each service.
- **timeout**: Timeout duration for requests.
- **retries**: Number of retries for failed requests.

---

## Services

The `services` section manages the services running for each flight controller.

- **services.service_name.enabled**: Boolean flag to enable or disable a specific service.
- **services.service_name.setting**: Settings specific to each service.

---

## REST API

The `rest_api` section defines settings related to the REST API.

- **port**: The port on which the API will run.
- **host**: The host address for the API. Use `0.0.0.0` to allow connections from everywhere.
- **global_prefix**: A prefix for the API routes.
- **as_https**: Boolean flag to enable or disable HTTPS.
- **ssl_keyfile_dir**: Directory path to the SSL key file.
- **ssl_certfile_dir**: Directory path to the SSL certificate file.

---

## Authentication

The `auth` section contains settings related to authentication.

- **enabled**: Boolean flag to enable or disable authentication.
- **jwt_secret**: A secret key generated by OpenSSL to encrypt tokens.
- **jwt_token_expire_minutes**: Duration (in minutes) for which a token is valid.
- **jwt_algorithm**: The algorithm used for JWT encryption.
- **users**: A list of valid users for authentication.

### User Fields

- **username**: The username of the user.
- **password**: The password of the user.
- **permission**: Permission level:
  - `11`: Read and write access.
  - `10`: Read-only access.
  - `0`: No access.
- **active**: Boolean flag to enable or disable the user.
- **is_admin**: Boolean flag to specify if the user has admin permissions.

---

This configuration file allows you to customize various aspects of the application, including flight controller settings, service management, API properties, and authentication.
